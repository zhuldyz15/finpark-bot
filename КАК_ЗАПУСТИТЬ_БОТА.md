# Telegram-бот оценки диагностик Finpark — как запустить

Партнёр пишет боту текст диагностики (или присылает файл .txt) и сразу получает в чате оценку: светофор по этапам, итоговый балл, сильные стороны, точки роста и рекомендации. Оценку ставит ИИ Claude — ключ API хранится только на сервере, партнёрам не виден.

В папке:
- `bot.js` — сам бот (без сторонних библиотек, нужен только Node.js 18+);
- `package.json` — запуск командой `npm start`;
- `.env.example` — пример переменных окружения.

---

## Шаг 1. Создайте бота в Telegram

1. В Telegram откройте **@BotFather**.
2. Команда `/newbot` → задайте имя и username бота (заканчивается на `bot`).
3. BotFather пришлёт **токен** вида `1234567890:AAE...` — скопируйте его.

## Шаг 2. Получите ключ Anthropic API

1. https://console.anthropic.com → **API Keys** → **Create Key** → скопируйте `sk-ant-...`.
2. Пополните баланс в **Billing** (оплата по факту, см. «Сколько стоит»).

---

## Шаг 3. Запустите бота на сервере (Railway — проще всего)

Боту нужен постоянно работающий сервер. Railway даёт это бесплатно на старте.

1. Загрузите папку `finpark-bot` в репозиторий на https://github.com (New repository → **uploading an existing file** → перетащите файлы).
2. Зайдите на https://railway.app, войдите через GitHub.
3. **New Project → Deploy from GitHub repo** → выберите репозиторий.
4. Откройте сервис → вкладка **Variables** → добавьте переменные:
   - `BOT_TOKEN` = токен от BotFather
   - `ANTHROPIC_API_KEY` = ваш `sk-ant-...`
   - `ACCESS_CODE` = кодовое слово для партнёров (необязательно, но рекомендую — например `finpark2026`)
   - `MODEL` = `claude-sonnet-4-6` для более точной оценки (необязательно; по умолчанию дешёвый `claude-haiku-4-5-20251001`)
5. Railway сам выполнит `npm start`. В логах появится «Finpark bot запущен» — готово.
6. Откройте своего бота в Telegram, напишите `/start`.

### Альтернатива — Render
https://render.com → **New → Background Worker** → подключите репозиторий → **Start Command:** `node bot.js` → добавьте те же переменные окружения.

### Запуск на своём компьютере (для теста)
```bash
cd finpark-bot
# Windows: set BOT_TOKEN=... затем set ANTHROPIC_API_KEY=...
export BOT_TOKEN="токен"
export ANTHROPIC_API_KEY="sk-ant-..."
export ACCESS_CODE="finpark2026"   # необязательно
npm start
```
Пока окно открыто — бот работает. Для постоянной работы используйте Railway/Render.

---

## Как пользуются партнёры

1. Открывают бота, пишут `/start`.
2. Если задан `ACCESS_CODE` — один раз присылают кодовое слово.
3. Вставляют текст диагностики одним сообщением (или присылают файл .txt) → получают оценку.

## Сколько стоит

Платите только Anthropic за оценки. На модели Haiku — доли цента за диагностику; на Sonnet дороже, но точнее. Тарифы: https://www.anthropic.com/pricing. Лимит расходов ставится в **Billing → Limits**. Railway/Render на старте бесплатны при небольшой нагрузке.

## Если что-то не работает

- В логах «Нет BOT_TOKEN» / «Нет ANTHROPIC_API_KEY» — не добавили переменные окружения (Шаг 3, п. 4).
- «Ошибка Anthropic API» — закончился баланс или неверный ключ.
- Бот не отвечает — проверьте, что сервис в Railway/Render запущен (статус Active) и в логах есть «Finpark bot запущен». Один бот-токен должен работать только в одном месте одновременно.

Логика оценки (этапы, веса, критерии, светофор) полностью совпадает с методичкой и Excel-шаблоном — это одна и та же система.
